# PRD: Screen Guardian 功能扩展

## 1. 产品概述

Screen Guardian 是 Windows 截屏录屏行为审计与管控系统。本次扩展增加以下核心能力：
- **规则自动匹配** — 按进程名/窗口标题自动应用防截屏保护
- **托盘常驻与自动监控** — 后台守护，实时感知新窗口并自动保护
- **GUI 界面** — 图形化管理窗口列表、规则和监控状态
- **威胁检测** — 实时检测截屏/录屏应用活动
- **审计日志** — 完整记录保护操作和系统事件
- **分层保护架构** — 多种保护机制自动回退，确保最大兼容性
- **GPU 截图防护** — 针对现代截屏工具（OBS、Game Bar）的高级保护

## 2. 目标用户

- 需要防止敏感窗口被截屏/录屏的个人用户
- 开发者、安全研究人员

## 3. 功能需求

### 3.1 规则自动匹配

**FR-RULE-01**: 规则模型
- 每条规则包含：ID、组ID、名称、进程名模式（支持正则）、是否保护、启用/禁用、优先级
- 规则存储为 JSON 文件（`rules.json`）

**FR-RULE-02**: 规则组管理
- 规则必须归属某个规则组
- 规则组支持 CRUD 操作
- 规则组可以包含 0~N 条规则

**FR-RULE-03**: 规则匹配引擎
- 枚举窗口后，对每条规则按优先级依次匹配进程名
- 首条匹配的规则生效（短路逻辑）
- 匹配时自动调用 `AffinityOrchestrator` 应用保护

**FR-RULE-04**: CLI 规则管理
- `rule list` — 列出所有规则
- `rule add` — 添加规则（进程名模式 + 保护状态）
- `rule remove` — 删除规则
- `rule enable/disable` — 启用/禁用规则

### 3.2 分层保护架构

**FR-PROTECT-01**: 分层回退机制
系统按优先级依次尝试多种保护方法，失败时自动回退：

| 层级 | 方法 | 说明 | 依赖 |
|------|------|------|------|
| Layer 1 | SetWindowsHookEx + DLL | Windows 官方支持，最安全 | DLL 文件 |
| Layer 2 | Shellcode 注入 | 无需额外文件，兼容性好 | 无 |
| Layer 3 | DLL 注入 | 经典方案，最终回退 | DLL 文件 |

**FR-PROTECT-02**: 保护方法状态追踪
- 记录每个窗口使用的保护方法
- 记录保护失败原因
- 提供保护方法统计信息

**FR-PROTECT-03**: 保护方法配置
- 可配置启用/禁用特定保护方法
- 可配置回退顺序
- 可配置超时时间

### 3.3 GPU 截图防护

**FR-GPU-01**: 截屏技术分类

| 类型 | 技术 | 代表工具 | 保护难度 |
|------|------|---------|---------|
| GDI 截图 | BitBlt, GetDIBits | 传统截屏工具 | 简单 |
| DXGI | Desktop Duplication API | OBS Display Capture | 困难 |
| Windows Graphics Capture | 官方新 API | Windows 截图工具 | 困难 |
| Game Capture | Hook DirectX/Vulkan | OBS Game Capture | 极难 |

**FR-GPU-02**: DXGI Desktop Duplication 防护
- Hook DXGI 的 `IDXGISwapChain::Present`
- 在帧被复制前返回黑屏/空帧
- 针对 OBS Display Capture 等工具

**FR-GPU-03**: Windows Graphics Capture API 保护
- 使用 Windows 官方的截屏保护机制
- 设置窗口属性禁止捕获
- 仅对使用新 API 的工具有效

**FR-GPU-04**: Game Capture 防护（高级功能）
- Hook Direct3D 9/11/12 的 Present
- Hook Vulkan 的 VkQueuePresent
- 在帧呈现前清空/加密帧缓冲
- 需要分别处理各图形 API

**FR-GPU-05**: GPU 保护配置
- 可配置启用/禁用 GPU 保护
- 可配置保护级别（基础/高级）
- 提供性能影响提示

### 3.4 托盘常驻与自动监控

**FR-DAEMON-01**: 系统托盘
- 托盘图标显示运行状态
- 右键菜单：显示主窗口 / 查看状态 / 暂停监控 / 退出

**FR-DAEMON-02**: 自动监控守护
- 后台轮询（可配置间隔，默认 3 秒）枚举窗口
- 新出现的窗口自动匹配规则并应用保护
- 已关闭的窗口从监控列表移除

**FR-DAEMON-03**: 开机自启
- 可选启用，写入注册表 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`

**FR-DAEMON-04**: 状态持久化
- 监控状态、已保护窗口列表持久化，重启后恢复

### 3.4 威胁检测

**FR-THREAT-01**: 截屏应用检测
- 维护已知截屏/录屏应用列表
- 实时检测这些应用的进程活动
- 提供威胁级别标识（高危/中危/低危）

**FR-THREAT-02**: 进程监控
- 监控当前运行的所有进程
- 支持立即扫描功能
- 提供进程详情查看

### 3.5 审计日志

**FR-LOG-01**: 事件记录
- 记录所有保护状态变更
- 记录规则匹配事件
- 记录系统启动/停止事件
- 记录威胁检测事件

**FR-LOG-02**: 日志管理
- 支持按级别过滤（INFO/WARN/ERROR）
- 支持按类型过滤
- 支持导出功能
- 支持清空功能

### 3.6 GUI 界面

**FR-GUI-01**: 窗口列表视图
- 表格展示所有可见窗口（进程名、PID、标题、保护状态、威胁标记）
- 支持排序、搜索过滤
- 一键切换单个窗口保护状态
- **自动刷新功能**: 可切换开关，支持选择间隔（3秒/5秒/10秒/30秒/1分钟）
- 状态持久化到 localStorage

**FR-GUI-02**: 规则管理视图
- 双面板布局：左侧规则组列表，右侧规则列表
- 规则组 CRUD 操作
- 规则 CRUD 操作
- 实时预览匹配结果

**FR-GUI-03**: 审计中心
- 监控开/关状态
- 已保护窗口计数
- 最近变更日志（保护状态变更记录）

**FR-GUI-04**: 威胁检测
- 显示已知截屏/录屏应用列表
- 进程监控状态
- 立即扫描功能

**FR-GUI-05**: 日志页面
- 系统事件日志记录
- 支持按级别、类型过滤
- 支持导出和清空

**FR-GUI-06**: 系统设置页面
- 监控间隔配置
- 开机自启开关
- helper EXE 路径配置
- 策略文件路径配置

**FR-GUI-07**: 关于页面
- 版本信息
- 开发者信息
- 技术栈说明
- 版权信息

**FR-GUI-08**: 底部状态栏
- 显示监控状态（运行中/已停止）+ 状态指示灯
- 显示已保护窗口数量
- 显示最后扫描时间
- 显示规则数量
- 显示版本号
- 每 5 秒自动刷新

## 4. 非功能需求

- **性能**: 监控循环 CPU 占用 < 1%，内存 < 50MB
- **兼容性**: Windows 10/11，x64 和 ARM64
- **安全性**: 分层保护架构，优先使用官方 API，回退到注入方案
- **可维护性**: 核心逻辑在 `screen-guardian-core`，GUI 和 daemon 独立 crate
- **可扩展性**: 保护方法可插拔，易于添加新的保护机制

## 5. 保护架构优先级

| 优先级 | 功能 | 保护范围 | 工作量 | 状态 |
|--------|------|---------|--------|------|
| **P0** | 分层回退机制 | GDI 截图 | 中 | 规划中 |
| **P1** | SetWindowsHookEx + DLL | GDI 截图 | 中 | 规划中 |
| **P2** | DXGI Hook | Display Capture | 大 | 规划中 |
| **P3** | WGC 保护 | 新 API 截屏 | 小 | 规划中 |
| **P4** | Game Capture 防护 | 游戏录屏 | 极大 | 未来

## 5. 不在范围内

- 远程管理/网络功能
- 多用户/权限管理
- 录屏内容检测（仅管控截屏行为）
